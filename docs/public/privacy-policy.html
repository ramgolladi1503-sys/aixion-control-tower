1. Who operates this app

Aixion Control Tower is a mobile approval console for reviewing, approving, rejecting, and auditing AI-assisted work execution. The app helps authorized users control work orders, approval requests, agent-created tasks, MCP queue items, sessions, invites, roles, and audit records.

2. Data we collect and process

Account data

We may process email address, display name, user ID, role, email verification state, invite acceptance metadata, session state, and account deletion request status.

We use this data for account creation, login, session management, role-based access control, invite acceptance, security, abuse prevention, admin access management, and account deletion request handling.

Approval, work-order, and operational content

We may process project IDs and names, project descriptions, work-order goals, work-order context, tasks, affected areas, required tests, rollback plans, approval titles, approval summaries, target branches, file change paths, diffs, proposed content metadata, repository names, branch names, validation commands, execution summaries, source/provenance metadata, MCP request metadata, MCP tool names, MCP arguments, and decision history.

This data is used to display approval requests, review proposed work, record decisions, show work-order provenance, and prove controlled AI-assisted execution.

Do not enter secrets such as API keys, passwords, private keys, database credentials, signing keys, or access tokens into work orders, approvals, MCP arguments, or agent task fields.

Audit and security data

We may process audit event type, actor identifier, entity ID, event details, timestamps, session revocation state, role changes, invite creation/acceptance/revocation/expiry records, approval decisions, failed or blocked request details, and account deletion request audit events.

Audit records support security traceability, approval evidence, admin review, incident investigation, compliance, abuse prevention, and accountability.

Device, session, server, and network data

We may process session ID, user ID, active/revoked session state, created and expiry timestamps, platform labels such as Android, app version when provided, device registration IDs if notifications are enabled, IP address, request timestamp, request path, HTTP status, user-agent/client metadata, server logs, and runtime errors.

This data is used for authentication, access management, service reliability, troubleshooting, security monitoring, abuse prevention, and incident investigation.

Push notifications, if enabled

If push notifications are enabled in a release build, we may process Firebase Cloud Messaging registration tokens, platform, app version, notification delivery status, and notification failure reason to send approval or operational notifications.

Data we do not intend to collect

Based on current product scope, the app is not intended to collect precise location, contacts, SMS/MMS, call logs, photos/videos/audio from the Android device, calendar data, health data, payment card data, advertising ID for ad targeting, browsing history, or children's data.

3. How we use data

  • Account management and authentication.
  • Session verification and access control.
  • Role and permission enforcement.
  • Approval review and decision recording.
  • Work-order management and source provenance.
  • MCP queue review and forwarding control.
  • Audit trail generation and security review.
  • Owner/admin access management.
  • Account deletion request handling.
  • Security, abuse prevention, troubleshooting, and service reliability.

We do not sell user data. We do not use app data for third-party advertising, cross-app behavioral profiling, or unrelated marketing.

4. Data sharing and service providers

The app sends data to the backend service configured for the deployment. The backend is necessary for account/session management, approvals, work orders, audit logs, account deletion request handling, and MCP/agent control.

Data may be processed by service providers that host or operate the backend, database, logging, deployment, notification, email, security, or monitoring systems on behalf of the app operator.

Possible providers include backend hosting, managed database, logging/monitoring, GitHub or repository integration if enabled, Firebase Cloud Messaging if notifications are enabled, and email delivery if verification or invite email delivery is enabled.

Data may also be disclosed if required for legal obligations, security investigation, fraud/abuse prevention, service operation, or organizational transfer with appropriate notice.

5. Admin and owner visibility

Because this is an approval-control console, authorized owner users may see operational and security information related to other users, including user email, display name, role, invite status, session metadata, active/revoked session state, audit events, approval/work-order actions, source/provenance metadata, and account deletion request audit status.

6. Security

Production and review environments should use HTTPS or equivalent modern cryptography for user data transmitted off-device. Local development or LAN demos may use HTTP only for controlled testing and must not be treated as production/review release configuration.

Sensitive data such as API keys, backend tokens, invite codes, keystores, signing passwords, FCM server keys, private keys, access tokens, or production database files must not be committed to source control or bundled in release artifacts.

8. Account deletion

Users can request account deletion through the public account deletion page:

Request account deletion

The app also provides an authenticated in-app request path from the Account screen. When an authenticated request is received, active sessions are revoked, account access is disabled, and an audit event is recorded.

Some audit/security records may be retained or anonymized where required for security, fraud prevention, legal obligations, abuse prevention, or compliance. Temporary account disabling, freezing, logout, or session revocation alone is not full account deletion.
